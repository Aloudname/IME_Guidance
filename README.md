## 引言

> “生物医学工程究竟是个什么样子，高教界搞了很多年，也并没有完全搞清楚。可能一些高校的思路比较好，搞了个智能医学工程，可惜后来也僵化了。”

这个仓库（repo）面向以 ~~神圣罗马帝国~~ **智能医学工程专业生** 身份踏入大学校园的学弟学妹们，为羽翼还未丰满、对未来还不甚了解的你们，提供求学、工作和生活上的微薄参考。与你们一样，初入校园时的我们也怀抱热情（但更多的是迷茫和疑惑）。作为 **“新兴专业”** 的前几批学生，我们成长求学的道路上少不了碰壁、挫折和困惑。如今大学四年生活已近尾声，各奔前程之时，我们对曾经的一些问题有了自己的答案，也逐渐发现了目前生涯规划中的不足。于是，我们想尽可能多地将自己来时路上求学和生活中的挫折和感悟记录下来，给你们讲述自己的 **“失败学”**。同时，我们想感谢四年时光里给予我们无限支持和关怀的老师和辅导员，感谢培育我们的山东大学控制科学与工程学院。

如果对你有用，请不吝收藏我们的 repo！另外，我们欢迎 IME / BME 专业的后辈们继续完善她，帮助更多后辈！

## 这个 repo 有什么？

如引言所述，我们提供一切有关求学、工作、生活上的微薄参考。我们将 repo 分成三个部分，将一些 **基础技能**（比如信息检索、GitHub 和 AI 工具的使用等）放在第一部分，将 **求学** 和 **工作** 放一起做第二部分，**生活** 做第三部分。

我重点分享的内容是 **求学** 和 **工作**。所谓的 **“新兴专业”** 不如传统的大类专业一般有明确的培育体系，很多事情需要学校、老师和学生摸索着来。正因此，我们不能仅依靠学校提供的培养方案来规划自己的路，那样会丢掉自身发展的主导权，而且会局限自己的视野。实际上，如果我们打开招聘软件一搜，就会发现目前的培养方案与市场需求是有一些出入的。另外，作为 **电子电气大类**（包括信号和信息处理、电力、电子、计算机、通信、控制、遥感、生物医学、系统智能化等）的学生，我们需要具备相当完备的计算机知识和素养。这些都是目前培育方案欠缺的。

对 **求学** 和 **工作**，我会结合自身的职业规划（AI 全栈）以及目前的专业培养方案来叙述，涉及 AI 的方方面面，为初入此门的后辈描绘一个整体的图景（以便解答最大的疑惑：“这个专业干嘛的，什么工作方向”），并在这个过程中向你们揭示培养方案和个人规划的异同。还是那句话，**一定不要完全依赖培养方案（当然也不要完全依赖我们的 repo），一定要有自己的打算**。

## 如果你也想做贡献

通过提交 `issues`（反馈）来指出 bug、提出需求，或者通过提交`pull requests`（合并请求）对仓库内容做改动，这些都有助于完善本 repo。开源社区鼓励这种所谓的 **“contribute”**。如果你想指出 bug、提出需求，可以直接在本项目的 `issues` 里提出意见！如果你有任何想分享的理解、经验、知识技能路线，乃至求学、求职、生活中的任何有意义的事情，都可以参考 [社区里的仓库.md](基础技能/2_社区里的仓库.md) 里关于 `fork`（创建分支）和 `pull requests`（合并分支）的内容，学习如何从本仓库创建属于自己的分支，修改自己的分支，并最终向本仓库提交  **自己的分支** 向 **本仓库** 的合并请求。

增加新内容时，请注意使用 Markdown 文本（`.md` 后缀。这种文本容易渲染，学起来也很快），将文档中用到的图片资源放在特定位置（`src/` 内），并保持本仓库风格的内容分区和分册哦！
